# Snapshot Pi 项目概览与团队分工文档

`Snapshot Pi` 是一款专为**开发者与终身学习者**打造的智能辅助学习系统。它基于 **Pi Agent** 智能体开发内核，深度融合了“启发式学习智能体”、可视化低代码工作流画布，并实现了创新的“双轨遗忘曲线”常青记忆知识库体系与 QQ 移动端个人助理。

---

## 1. 项目介绍与核心目标

### 1.1 项目背景与定位
在信息爆炸的时代，主动学习与知识内化变得愈发困难。普通的笔记软件仅停留在“收集”阶段，而传统的 AI 助手则倾向于直接给出答案，剥夺了学习者主动思考的过程。

`Snapshot Pi` 的定位是**“第二大脑”与“启发式导师”的结合体**。它不单是一个工具，而是一个陪伴用户成长的学习生态系统。系统通过启发式的对话引导用户深挖概念，并自动将学习成果沉淀到双链知识库中，通过科学的遗忘曲线算法（SM-2）安排复习，实现知识的闭环管理。

### 1.2 核心系统架构
项目采用一体化的多包管理模式，包含以下四个核心功能层：
* **前端交互层 (Frontend)**：基于毛玻璃（Glassmorphism）镜面流光视觉风格，提供多窗口拖拽分栏的卡片式工作区，以及低代码工作流画布。
* **后端网关层 (Backend)**：提供统一的数据服务和多会话管理，并利用 Socket.io 实现高实时性的双向消息流。
* **智能体内核 (Pi SDK)**：驱动智能体的逻辑编排、工作流技能动态编译与热重载。
* **移动辅助端 (QQ Bot)**：通过嵌入式协议适配器实现与个人 QQ 账号的绑定，提供随时随地的移动端对话、自测与学习监控。

## 2. 核心系统功能 (V0.1 已完成)

系统目前已实现以下核心产品功能：

### 2.1 启发式 AI 对话与模型管理
* **启发式教学导师**：AI 不会直接给出代码或标准答案，而是以提问和引导的方式，帮助用户自主推导并掌握知识点。
* **多会话管理与预设工厂**：支持自由创建、切换和删除多个学习会话。内置“启发式导师”与“代码专家”等不同预设角色，支持灵活选择不同的大模型。
* **智能多模态拦截**：在聊天中上传图片时，系统会自动启用多模态子智能体提取图片信息，并静默补充为对话上下文，从而突破主模型的视觉输入限制。

### 2.2 可视化低代码工作流画板
* **可视化技能画布**：提供图形化拖拽连线画板，用户可以通过连接不同节点来搭建复杂的自动化学习或执行流程。
* **一键编译与热重载**：画布中的图形化工作流可一键编译为智能体标准技能文件，并实现即时热加载生效。
* **自我上演演化**：支持 AI 智能体根据用户交互反馈，自动生成或修改自身的可视化工作流。

### 2.3 双轨遗忘曲线常青知识库
* **智能置信度评估**：使用指数衰减遗忘模型实时评估用户对知识概念的掌握程度，并根据复习互动进行置信度提升。
* **自动归档与链接重写**：当概念置信度随时间衰减至临界值以下时，自动触发归档机制，并智能重写知识库中的双向链接，保持核心知识网的“常青”。
* **SM-2 间隔重复复习**：结合 SM-2 算法对用户的个人整理笔记进行复习调度，提供置信度徽章反馈。

### 2.4 QQ 个人助理 (移动端学习闭环)
* **公式渲染与分段发送**：在 QQ 私聊中支持复杂的 LaTeX 数学公式，系统会自动将其渲染为图片并附带文本发送，保障阅读体验；长文本消息可感知段落边界自动分段。
* **聊天对话沉淀**：自动提炼 QQ 聊天过程中的高价值知识信息，自动在后台生成对应的 Wiki 概念卡片。
* **移动自测系统 (Quiz)**：根据用户知识库中最薄弱的知识点，主动出题自测，作答结果直接反馈至遗忘曲线模型中，并根据表现奖惩学习经验值 (XP)。
* **数据仪表盘与周报**：定期整理自测打卡趋势和薄弱点分析，生成简洁美观的自测学习周报。

## 3. 团队分工规划 (4 人，均承担代码工作)

为保障项目在前端视觉、AI 内核、学习闭环三个维度上全面推进，4 名团队成员的分工划分如下：

### 3.1 开发者 A —— AI 交互与核心算法 (AI & Core Algorithm)
* **主要职责**：负责智能体内核的对接封装、AI 模型适配及核心算法设计。
* **功能开发内容**：
  1. **多智能体 (Sub-Agent) 编排框架**：开发串行、并行、监督及路由等多种多代理协作模式。
  2. **启发式交互逻辑**：设计系统提示词拦截注入机制，优化智能体启发式对话的稳定度。
  3. **指数衰减与复习算法**：演进并调优双轨知识库的掌握度评估数学模型。

### 3.2 开发者 B —— 后端业务与消息网关 (Backend & Messaging Gateway)
* **主要职责**：负责网关服务搭建、多渠道消息路由、数据持久化及 QQ 机器人适配。
* **功能开发内容**：
  1. **QQ Bot 移动适配与加固**：负责限流控制、心跳保活及公式服务端截图渲染服务。
  2. **会话与预设管理**：开发会话创建、删除及历史会话多端同步的后端逻辑。
  3. **数据统计与自测报告**：设计自测题库生成接口以及学习经验值 (XP) 统计和周报导出接口。

### 3.3 开发者 C —— 前端界面与可视化交互 (Frontend & Visualization)
* **主要职责**：负责前端高保真界面开发、拖拽画布交互、以及知识网络可视化。
* **功能开发内容**：
  1. **毛玻璃 (Glassmorphism) 主题重构**：实现黑暗/明亮双主题、弥散流光动态背景以及卡片磨砂质感。
  2. **2D 知识图谱**：开发基于双链拓扑关系的交互式力导向粒子图谱，直观呈现置信度与双链关系。
  3. **画布节点与 AI 协写**：在画布中扩充 API 请求、条件分支等控制节点，并在配置抽屉内开发 AI 协写气泡。
  4. **聊天控制台体验升级**：支持流式 Markdown 解析、代码块一键复制，实现输入框高度自适应。

### 3.4 开发者 D —— 系统集成与质量保障 (Integration & Quality Assurance)
* **主要职责**：负责工作流拓扑编译、系统集成部署、自动化测试及质量工程。
* **功能开发内容**：
  1. **低代码拓扑编译器**：开发并维护工作流到技能文件的拓扑排序编译与热加载重载闭环。
  2. **自动化测试套件**：编写核心算法（SM-2、置信度）的单元测试，以及主业务流的端到端（E2E）功能测试。
  3. **自动环境对齐**：开发多环境自适应对齐工具。

## 4. 自适应环境适配 (QQNT 与 NapCat 版本自动对齐)

为了解决项目迁移到其他机器、或系统 QQNT 静默升级时，因核心二进制不匹配导致的闪退或版本风控风险，项目在 `scripts/` 下提供了一键自动对齐工具：[sync-qq-shell.ps1](file:///c:/Users/lisky/Desktop/projectEL/scripts/sync-qq-shell.ps1)。
* **核心功能**：自动从注册表中读取本地安装的 QQ 客户端路径与版本，校验其是否满足 `>= 9.9.15` 最低运行要求；自动将适配的官方核心二进制模块同步复制到本地项目目录中，并全自动重写所有配置文件，5 秒内实现环境的无感适配。
